# 4Dpapers v0.1.2 — Release Notes (draft)

Prepared: 2026-07-16 · Branch: `launch-prep-v0.1.2` · Supersedes the unannounced `v0.1.1` tag.

v0.1.2 is the first **publicly announced** release. It is v0.1.1 plus an
offline-hardening patch: the dashboard UI and PDF export now work with no network
access, which the `0.1.1` image did not. Same feature scope otherwise.

---

## GitHub Release body (paste into the `v0.1.2` release)

**4Dpapers v0.1.2 — interactive 3D figures for scientific papers, now offline-safe**

Write a paper in Quarto Markdown, drop in a simulation result, and get an
interactive 3D figure in HTML and a static figure in PDF — all from a local IDE
in one browser tab.

This release makes the app work in air-gapped and egress-restricted containers
(HPC, hospital, and secure research networks), where the previous build silently
broke:

- **Dashboard renders offline.** Tailwind, CodeMirror, the Phosphor icons, and the
  Outfit / JetBrains Mono fonts are now bundled in the image instead of loaded
  from CDNs, so the editor no longer collapses to unstyled HTML without internet.
- **PDF export is network-free.** Exporting a paper that references an unreachable
  host no longer hangs with no error; remote assets are skipped and the PDF still
  renders. Set `FOURD_PDF_ALLOW_REMOTE=1` to opt back into fetching them.
- **Real export tests in CI.** Compile-to-HTML, standalone-HTML export, and
  PDF export now run the actual Quarto + WeasyPrint pipeline on every pull request,
  so a silent export regression can't ship again.

### Run it

```bash
git clone https://github.com/4dpapers/4dpapers.git
cd 4dpapers
cp .env.example .env
IMAGE=ghcr.io/4dpapers/4dpapers:0.1.2 FOURD_WORKSPACE=./examples/niederer docker compose up
```

Open http://localhost:5006 — the example cardiac paper loads with its 3D figures
live. Full changelog: `CHANGELOG.md`.
