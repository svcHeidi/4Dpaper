# 4Dpapers v0.1.1 Launch Readiness

Audit date: 2026-07-15
Owner: Simão Nieto de Castro

## Decision

The local v0.1.1 candidate is technically approved for a GitHub-first release.
The public release remains **on hold** only until the candidate is committed,
reviewed in CI, published as a multi-architecture image, and tested from a fresh
clone. The deferred marketing website is not a v0.1.1 gate.

## Evidence recorded

- Python regression: `403 passed, 11 skipped`.
- Final local image: `4dpapers-0.1.1-rc-final`, native `arm64`, with image label
  and `/app/VERSION` both reporting `0.1.1`.
- Production smoke: non-root startup, authenticated APIs, HTML compilation, PDF
  export, route protection, and normal-mode Quick routes returning 404.
- Quick smoke: isolated VTK preview/export, exactly two retained HTML files,
  read-only source mount, writable output mount, and complete cleanup.
- Manual full mode: compilation, HTML/PDF exports, figure rotation, field
  switching, time playback, and synchronized panels pass.
- Manual Quick mode: Niederer VTK-series and a real OpenFOAM cavity case pass.
- OpenFOAM evidence: `U` and `p`, six timesteps (`0` through `0.5`), live field
  playback, standalone HTML after server shutdown, and no browser page errors.
- OpenFOAM isolation: source directory mounted read-only; exactly
  `fig-case.html` and `fig-case-standalone.html` retained; temporary workspace
  and container removed on shutdown.
- Docker/Quarto: the Niederer paper renders inside the app image; browser audit
  found 10 time-series frames, 7 live canvases, no overflow, and no page errors.
- Current public root demo returns 200 and is the link used by the public README.
  Deferred `/demo/` and `heart-demo.html` paths are not claimed for v0.1.1.
- Marketing website work is preserved locally on `website-v0.1.2` at `de7acfd`.

## External go-live sequence

1. Review and commit the candidate; confirm a clean worktree and record the SHA.
2. Push a PR and require regression, image-smoke, and existing example-preview
   checks.
3. Merge, verify the root demo remains healthy, then tag `v0.1.1`.
4. Wait for both GHCR architectures and record the published digest.
5. From a fresh clone, run the exact README command using image `0.1.1`.
6. Publish GitHub Release notes and use the repository plus root demo in the
   founder's LinkedIn announcement.

## After launch

- Prioritize installation failures and regressions in the verified format set.
- Replace the browser-readable shared key with an HttpOnly session design before
  any broader hosted or multi-user product.
- Vendor or pin browser dependencies currently loaded from CDNs.
- Resume the marketing website from `website-v0.1.2` and audit the fully
  assembled Pages artifact before deploying it in a later release.
