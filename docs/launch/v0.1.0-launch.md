# v0.1.0 Launch Kit

Demo-led launch. Goal: **real users + feedback**, under the author's real name, proudly.
Low-risk vehicle: a static WebGL demo on GitHub Pages — no hosted backend, no upload/auth surface.

---

## Launch order

1. Create the `4dpapers` GitHub org (Free plan).
2. Transfer the repo to `4dpapers` (Settings → Transfer ownership). Optionally rename `4Dpaper` → `4dpapers`.
3. Deploy the demo: `bash deploy/pages-deploy.sh`, then enable Pages (Settings → Pages → `gh-pages` / root).
   - Live URL: `https://4dpapers.github.io/4dpapers/`
4. Public-surface check — view what a stranger sees, on desktop **and** phone:
   repo landing page · README first screen · Pages demo · release page.
5. Tag and publish the release (notes below).
6. Post to LinkedIn (draft below). Put the link in the **first comment**, not the post body.
7. Reply to every comment/DM in the first ~2 hours.

Not now: forming a limited company; hosting the full Docker app publicly. Revisit both only on real traction.

## GitHub Release notes — v0.1.0

**4Dpapers v0.1.0 — interactive 3D figures for scientific papers**

Write your paper in Quarto Markdown, embed simulation and mesh data as **live, rotatable WebGL
figures**, and export to HTML or PDF from one local Docker app. No plugin zoo, no manual screenshotting.

**What works today**
- Interactive 3D figures with field switching (`U`, `p`, `T`, `Vm`, …) and timestep playback
- Synchronized multi-panel and time-series layouts with shared cameras
- HTML export (self-contained, opens anywhere) and PDF export
- **Verified formats:** VTK/VTU/VTP/PVD/VTK-series, XDMF+HDF5, STL/OBJ/PLY, Gmsh/MED, Plotly JSON
- **Externally validated:** OpenFOAM `.foam`
- **Early/experimental:** Exodus, CGNS, EnSight, Abaqus `.inp` — implemented, feedback wanted

**Try it:** live demo → https://4dpapers.github.io/4dpapers/
**Run it:** `docker compose up` (see README)

This is an early release. It already renders real scientific and engineering datasets — I'm looking
for users to shape what comes next. Issues and DMs welcome.

## Why these choices

- **Link in first comment, not post body** — LinkedIn suppresses reach on posts with external links.
- **Narrow ask** — "10 minutes, these specific tools" converts far better than "check it out."
- **Honest maturity split** — naming what's verified vs experimental builds trust with researchers and
  pre-empts "it broke on my file" disappointment.
- **Native media** — attach a ~30s screen recording of a rotating/animating figure to the post itself;
  native video outperforms link previews.
