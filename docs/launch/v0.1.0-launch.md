# 4Dpapers v0.1.1 GitHub Release Runbook

Last revised: 2026-07-15

## Release decision

Launch v0.1.1 from GitHub. Keep the existing interactive paper at
`https://4dpapers.github.io/4dpapers/` as the live demo.

The new marketing landing page, embedded heart showcase, SEO files, and `/demo/`
site layout are deferred to v0.1.2. They are preserved on the local
`website-v0.1.2` branch at commit `de7acfd`; they are not v0.1.1 release gates.

## v0.1.1 scope

- GitHub repository, tagged GitHub Release, and release notes.
- Tagged `amd64` and `arm64` GHCR image.
- Full paper-authoring mode as the primary workflow.
- Isolated Quick Export as a clearly labelled beta in the same image.
- Existing root Pages demo, rebuilt by the established example workflow.
- Founder-led LinkedIn announcement linking to the repository and root demo.

Out of scope:

- The new marketing landing page and `/demo/` URL structure.
- A publicly hosted dashboard or multi-user service.
- Presenting the source-available license as open source.
- Presenting experimental readers as fully verified.

## Required go/no-go gates

1. The candidate diff is reviewed, committed, and clean at a recorded SHA.
2. PR regression, image smoke, and existing example-preview checks pass.
3. Tag `v0.1.1` produces both `amd64` and `arm64` images; never move `v0.1.0`.
4. A fresh clone starts the published `0.1.1` image with the exact README command.
5. The existing root live demo returns 200, creates live canvases, and has no
   uncaught browser errors after the release merge.
6. README and launch copy link to the root demo, not deferred `/demo/` or
   `heart-demo.html` paths.
7. Public wording says **source-available**, identifies Quick Export as **beta**,
   and separates verified, manually validated, and experimental formats.
8. The previous `v0.1.0` tag and image digest are recorded for rollback.

## Release sequence

1. Review and commit the v0.1.1 candidate on `launch-prep-v0.1.0`.
2. Push the branch and open a PR; wait for every required check.
3. Merge the reviewed SHA and verify the root Pages demo remains healthy.
4. Create tag `v0.1.1` and wait for the multi-architecture GHCR publication.
5. Test the README command from a fresh clone using the published tag and record
   the image digest.
6. Publish the GitHub Release notes.
7. Record a short native demo video and publish from the founder's personal
   LinkedIn profile. Link to the GitHub repository/release and root live demo.
8. Monitor installation failures, format regressions, Discussions, and issues.

## Accurate public claims

Use:

> 4Dpapers is a source-available, local research-paper IDE that turns supported
> simulation data into interactive 3D HTML figures and static PDF figures.

Evidence tiers:

- Verified locally: VTK/VTU/VTP/PVD/VTK-series, XDMF + HDF5, STL/OBJ/PLY,
  Gmsh/MED, generic HDF5, and Plotly JSON.
- External/manual validation: OpenFOAM.
- Experimental: Exodus, CGNS, EnSight, and Abaqus.

License:

> Free for non-commercial research and education under PolyForm Noncommercial
> 1.0.0. Commercial use requires a separate license.

## Rollback

- Docker: pin `ghcr.io/4dpapers/4dpapers:0.1.0` or its recorded digest.
- Repository: revert the v0.1.1 merge commit without moving either release tag.
- Demo: rerun the existing Example Preview Pages workflow at the rollback SHA.
- Website: do not deploy `website-v0.1.2` as part of this release.

## First 72 hours

- T+2h: record substantive replies, datasets offered, and installation failures.
- T+24h: publish a concise known-issues note if a repeatable pattern appears.
- T+3d: prioritize broken installs and failures in the verified format set.
- Ship fixes as explicit patch releases; do not silently redefine an existing tag.
