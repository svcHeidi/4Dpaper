# Design: Vector PDF Export + Auto-Updating Paper-View HTML

**Date:** 2026-04-01
**Status:** Approved

---

## Problem

1. **PDF figure quality/sizing**: Quarto PDF export embeds rasterized PNGs (`1920×1080`, 16:9). The HTML preview renders figures at `900×600` (3:2). Different aspect ratios mean the PDF never looks like the HTML, and raster at print resolution is softer than vector.

2. **Slow iteration loop**: To check if a figure looks right in the PDF the user must click "Export PDF" and wait 60s+. There is no fast way to preview the paper layout.

---

## Goals

1. PDF figures use the existing per-figure `.pdf` (vector) files that PyVista's `save_graphic` already produces — crisp at any print size.
2. Panels in PDF use LaTeX minipage grids, each cell containing a subfigure vector PDF.
3. HTML and PNG aspect ratios are unified (both 3:2), so the paper-view preview matches the PDF exactly.
4. A new "Paper View" tab in the dashboard shows a static paper-like HTML that auto-rebuilds every 30 s while the user is active.

---

## Architecture

### 1. Vector PDF figures in Lua (`shortcodes.lua`)

**Individual figures (`4d-image`, `4d-pvsm`, `4d-video`):**

- Lua already branches on `quarto.doc.is_format("pdf")`.
- Change: when PDF, embed `state/figures/<id>.pdf` via `pandoc.Image` with `.pdf` extension instead of `.png`.
- LaTeX (`\includegraphics`) accepts `.pdf` natively — no LaTeX changes needed.
- Fallback: if `.pdf` does not exist (e.g. video frame), fall back to `.png`.

**Panel figures (`4d-panel`, `4d-timeseries`):**

- Currently Lua embeds a single composite `.png`.
- Change: emit a raw LaTeX block with a minipage grid.
  - `\begin{figure}` wraps `N` `\begin{minipage}{<1/ncols>\textwidth}` blocks.
  - Each minipage contains `\includegraphics[width=\linewidth]{state/figures/<sub_id>.pdf}`.
  - Caption goes below the whole figure environment.
- Rows are achieved by inserting a `\\[4pt]` between row groups of minipages.
- The composite `.png` (used by paper-view HTML) is still generated by Python — Lua only skips it for PDF.

### 2. Aspect ratio unification (`4dpaper.py`)

- `generate_png_figure`: change `window_size=(1920, 1080)` → `window_size=(900, 600)`.
- `generate_html_figure`: already `window_size=(900, 600)` — no change.
- `generate_video_figure` frame PNG: change `window_size=(1920, 1080)` → `window_size=(900, 600)`.
- Panel composite PNG (`generate_panel_png`): each cell is now 900×600; composite size = `(900*ncols, 600*nrows)` — computed dynamically from the first subfigure (already done, no change needed).
- HTML replacement in `generate_html_figure`: `100vw` → `900px`, `100vh` → `600px` — already correct.

### 3. Paper-view Quarto profile

**New file `_quarto-paperview.yml`:**

```yaml
format:
  html:
    theme: default
    toc: false
    embed-resources: false
    css: _extensions/4dpaper/paperview.css
    output-file: analysis_report-paperview.html
```

**New file `_extensions/4dpaper/paperview.css`:**

```css
body { max-width: 700px; margin: 2rem auto; font-family: Georgia, serif; font-size: 11pt; }
h1, h2, h3 { font-family: system-ui, sans-serif; }
figure img { width: 100%; }
```

**Lua detection:** `FOURD_PAPER_VIEW=1` environment variable.

- When set, `4d-image` / `4d-panel` / `4d-timeseries` / `4d-pvsm` shortcodes embed `<img src="state/figures/<id>.png">` directly (no iframe, no srcdoc).
- Panel shortcodes embed the composite `<id>.png`.
- All figures are served as local static files (no embed-resources), so build is fast.

**`run_quarto_render` in `dashboard/utils.py`:** add `output_format="paperview"` branch that sets `FOURD_PAPER_VIEW=1` and passes `--profile paperview`.

### 4. Dashboard Paper View tab (`dashboard/pages/paper_page.py`)

**Activity tracking:**

- A JS heartbeat snippet (injected into the dashboard HTML once) fires `postMessage({type:"fourd-heartbeat"})` every 10 s.
- A Tornado endpoint `POST /heartbeat` (or Panel's existing WS) updates `last_active` timestamp on `PaperPage`.

Simpler alternative (no extra endpoint): use Panel's existing `pn.state.execute` / periodic callback — track activity via a `pn.widgets.Button` or any user interaction. Actually, simplest: the 30s timer always fires; if the paperview iframe has been viewed at least once (user opened the tab), rebuild. If the tab was never opened, skip.

**Decision:** Gate rebuilds on whether the user has ever clicked to the Paper View tab in the current session (not idle-timeout). Simpler, no JS heartbeat needed.

**`PaperPage` additions:**

- `_paper_view_enabled: bool` — set to `True` when user first opens Paper View tab.
- `_paper_view_cb: PeriodicCallback` — 30 s interval, starts when tab first opened.
- `_paper_view_iframe: pn.pane.HTML` — shows the paperview HTML output.
- `_paper_view_building: bool` — skip tick if build already running.

**Periodic callback logic (`_tick_paper_view`):**

```python
def _tick_paper_view(self):
    if self._paper_view_building or self.is_building:
        return
    self._paper_view_building = True
    threading.Thread(target=self._run_paper_view_build, daemon=True).start()
```

**Build completion:** on exit code 0, update `_paper_view_iframe` with cache-busted `/output/analysis_report-paperview.html?t=<ts>` (Quarto output filename derived from profile name, or set explicitly in `_quarto-paperview.yml` via `output-file`).

**Tab layout:** The existing Paper tab gets a horizontal split or a tab switcher between "Interactive HTML" and "Paper View". Since Panel is already used, use `pn.Tabs` with two panes.

## Out of Scope

- Shortcode `width=` / `height=` attributes (can be added later).
- Playwright/headless browser screenshots.
- Composite vector PDF stitching for panels (LaTeX minipages handle this).
- Any changes to the camera sync or field switching logic.
