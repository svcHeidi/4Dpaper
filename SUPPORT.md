# Support

4Dpapers is an early, founder-maintained project without a support SLA.

- Use [GitHub Discussions](https://github.com/4dpapers/4dpapers/discussions)
  for setup questions, format guidance, and workflow ideas.
- Use [GitHub Issues](https://github.com/4dpapers/4dpapers/issues) for
  reproducible bugs and format-compatibility reports.
- Follow [SECURITY.md](SECURITY.md) for vulnerabilities; do not report them in
  a public issue.

For a useful report, include the 4Dpapers version, operating system and CPU,
Docker version, input format, exact command, relevant logs, and the smallest
redistributable sample you can share. Remove credentials, private paths, and
unpublished data first.

Commercial licensing and contracted support are separate from community
support. Until a dedicated commercial contact is published, start a Discussion
without sharing confidential details.
